# Welcome! 💙

We’re glad you’re here, and that you’re thinking about contributing to our repo. As the maintainers, we understand 
that AI-enabled workflows are ubiquitous now, and we don't forbid the use of AI. In fact, we’d like to help you use 
it constructively and with a sense of community and responsibility.

## Usage

Our team uses AI for prototyping features, code review, and generating code along pre-existing patterns. 
We want to make our expectations for AI usage clear. Here’s what we ask if you use AI while contributing 
to this project.

First, please review the project's [CONTRIBUTING.md](./CONTRIBUTING.md) for guidance on submitting pull requests.

Second, we have used Claude in this codebase. However, AI is rapidly evolving across a plethora of tools, 
and if you use a different tool, we recommend copy/pasting our configuration markdown files into your chat 
interface and asking your model to generate instructions based on that. Take a look at [CLAUDE.md](./CLAUDE.md) and 
the [.claude folder](./.claude) as they include valuable information such as testing best practices,
style preferences, and the like.

We have a folder of [repo-specific skills](./.claude/skills) for you to use with Claude, please avail yourself of them!

## Avoiding Slop

As we are each responsible for the code we contribute, we are also responsible for respecting reviewers' 
time and avoiding AI slop. We consider slop to be:

- Massive PRs created without empathy for the reviewer, often containing long, bullet-pointed PR descriptions
- Code that creates new patterns or functions, without having supported reasoning or considering pre-existing patterns or functions
- Excessive comments describing what the code is, instead of why it was implemented a certain way
- A PR that is not attached to a specific pre-existing open ticket or new ticket that isn’t well defined

If we have determined a pull request, issue, or discussion post falls into this category, **it will be closed**.
